検証作業について

2022年4月19日


現時点で特定のテーマでの検証作業はありません。
今後発生するであろう検証作業の事前調査的な要素として、以下のようなことを実施していきたいと考えています。

項目が多いかもしれません。これを全部すぐにやらなければならないというものではありません。復帰後も含めて今後こうした内容をベースに次第に領域を膨らませていければというものですのであせらず、ご自身のできるものからご検討ください。

検証対象
Azure Functions
    Functions
    Durable Functions
Logic Apps
Azure Kubernetes Service
App Service（若干優先度低）

上記の環境は、基本的な環境を構築できるようにしておきたいと考えています。これらの基礎知識は理解していただきたいと思います。
また、これらの環境上に簡単なアプリケーションをデプロイできるようにしておいていただきたいと思います（App ServiceとAzure Functiosnについては、Zipデプロイとコンテナデプロイの両方のやり方で出来ることが望ましいです）
とくにFunctionsは経験もあると思いますのでご自身の理解の幅を広げていただきたいと思います。AKSについてもアプリケーション開発者の視点で理解を進めていただきたいと思います。
（これらの環境の特徴などを理解していただき、簡単なレベルでのシステム設計ができる状態にあるのが望ましいと考えています。）

加えてアプリケーションから何らかのデータベースやストレージアクセスがあるような機能があるとベストです。（そうした要素を加えることでFunctionsやAKSに対する考慮事項も増えると思います）

ビルドパイプライン
Azure DevOps Pipeline
GitHub Actions

上記については、基本的な利用方法については理解をしておいていただきたいと思います。

アプリケーション言語
コンパイル系言語とスクリプト系言語それぞれ１つ以上で検証が実施できることが望ましいと考えています。またはコンパイル系／スクリプト系言語の環境で考慮すべき項目が把握できればよいです。（例えばDevOps Pipelineなどで、どうテストするか、どうビルドするか、どうデプロイするか、などです）


作業の記録
何らかの形で作業の記録を残していただきたいと思います。ブログなどの形式にしていただくのがベターです。
